# 问题排查指南

本文档汇总了实际使用中遇到的所有问题及解决方案，供配置新 OpenClaw 时参考。

---

## 问题1：Chrome 下载文件没有扩展名

**现象：** 点击下载后，Downloads 文件夹里没有文件，或文件没有扩展名无法打开。

**原因：** Gemini 生成的图片下载时，Chrome 会先把文件存入内部缓存（`Cache_Data` 目录），然后再移动到 Downloads。但文件存在缓存时会被 Chrome 锁定，且文件名没有扩展名。

**解决方案：**
使用 Chrome 缓存监控脚本（`wait_download_cache.py`），从缓存目录直接提取文件，通过文件头判断类型：

```python
with open(cache_file, 'rb') as f:
    header = f.read(16)
if header[:8] == b'\x89PNG\r\n\x1a\n':
    ext = '.png'  # PNG
elif header[:3] == b'\xff\xd8\xff':
    ext = '.jpg'  # JPEG
```

**注意：** 不要试图修改 Chrome Preferences 的下载目录设置，Chrome 会忽略这个设置。

## 问题2：Python 下载 lh3.googleusercontent.com 链接 403

**现象：** 用 Python 的 urllib 下载图片链接，返回 403 Forbidden。

**原因：** Google CDN 的图片链接包含浏览器会话认证（cookie/token），纯 Python 请求没有这个认证头，CDN 拒绝访问。

**解决方案：**
1. **主要方案**：用 Chrome 缓存提取（最可靠）
2. 用 Chrome 分享链接打开新页面后提取图片 URL
3. 永远不要试图绕过认证直接下载 CDN 链接

---

## 问题3：Chrome 缓存文件被锁定无法读取

**现象：** 缓存目录里的文件正在被 Chrome 使用，Python 报"文件被占用"。

**原因：** Chrome 下载时锁定了缓存文件。

**解决方案：**
用 `shutil.copy` 而非 `shutil.move`，copy 会读取文件内容而不需要移动文件。文件在缓存中保持不变，copy 出来的副本可以正常使用。

```python
import shutil
with open(newest, 'rb') as f:
    data = f.read()
with open(out_path, 'wb') as f:
    f.write(data)
```

## 问题4：Chrome 与 Playwright 不能同时使用同一个 Profile

**现象：** 用了 Playwright 控制 Chrome 后，OpenClaw 的 browser tool 连接不上。

**原因：** Chrome 的 remote debugging 模式只允许一个 debugger 连接。同一个 Profile 被 Playwright 连接后，browser tool 无法再连接。

**解决方案：**
使用**两个独立的 Chrome Profile**：
- `OpenClaw` profile → 专给 OpenClaw browser tool 使用
- `Playwright` profile → 给 Playwright 脚本专用

日常用 OpenClaw 的 browser tool 就足够，不需要 Playwright。

## 问题5：Chrome Profile 隔离问题

**现象：** 每次重启 Chrome，之前登录的 Gemini 状态消失了。

**原因：** 如果使用临时目录或不带 `--user-data-dir` 启动 Chrome，Chrome 会使用默认临时配置，每次关闭后数据丢失。

**解决方案：**
使用固定的 `--user-data-dir` 参数指定专用目录，并保证每次都用同一个目录启动：

```powershell
--user-data-dir=%USERPROFILE%\AppData\Local\Google\Chrome\OpenClaw
```

首次登录后，Chrome 会记住登录状态，之后启动自动恢复。

## 问题6：下载大文件（8MB+）需要等待很久

**现象：** 点下载后立刻检查 Downloads，没有文件，以为下载失败了。

**原因：** 完整尺寸图片（8-10MB）下载需要 1-3 分钟。

**解决方案：**
等待文件大小**稳定**（连续两次检查大小不变）再认为下载完成。缓存监控脚本已内置这个逻辑：

```python
# 等待大小稳定
for _ in range(60):  # 最多等5分钟
    time.sleep(5)
    size_now = newest.stat().st_size
    time.sleep(2)
    size_later = newest.stat().st_size
    if size_now == size_later and size_now > 10000:
        break  # 下载完成
```

## 问题7：snapshot 后元素引用（ref）发生变化

**现象：** 第一次 snapshot 显示 e10 是制作图片按钮，但点击后页面没反应，再 snapshot 发现 e10 变成了别的东西。

**原因：** Gemini 页面加载是动态的，每次页面状态变化（如弹出对话框、新内容加载），DOM 会重新渲染，元素引用会改变。

**解决方案：**
每次操作后都要重新 snapshot 获取最新的元素引用，不要依赖之前记住的 ref 值。特别是在：
- 点击"制作图片"后
- 发送请求后
- 页面加载完成后
都要重新 snapshot。

## 问题8：Python PowerShell 输出乱码

**现象：** 运行 Python 脚本时，PowerShell 输出中文全是乱码（GBK 解码失败）。

**原因：** PowerShell 默认编码是 GBK，Python 默认输出 UTF-8，编码不匹配。

**解决方案：**
在 Python 脚本开头加：
```python
import sys
sys.stdout.reconfigure(encoding='utf-8')
```
或者直接忽略这个问题，只关注脚本最后的输出结果（文件是否正确生成）。

## 问题9：MiniMax MCP 图片审核超时

**现象：** 调用 `mcporter call minimax.understand_image` 处理大文件（8MB+）时超时。

**原因：** 大图片上传和处理需要较长时间，超出 MCP 默认超时。

**解决方案：**
1. 先压缩图片再审核：
```python
from PIL import Image
img = Image.open('gemini_output.png')
img = img.convert('RGB')
img.save('temp_small.jpg', quality=80)
# 用 temp_small.jpg 做审核
```
2. 或者直接用截图方式（browser screenshot）获取图片，用 image 工具分析截图。

## 问题10：Chrome 显示"工具按钮是灰的"

**现象：** 打开 Gemini 后，工具按钮（制作图片、创作音乐）显示为灰色，不可点击。

**原因：** Google 账号未登录，或者账号没有 Gemini 图片生成权限。

**解决方案：**
1. 在 Chrome 中登录 `https://gemini.google.com`
2. 确认账号有 Gemini 权限（普通 Google 账号即可）
3. 如果仍然灰色，清除 Cookie 后重新登录：
   - 地址栏输入 `chrome://settings/clearBrowserData`
   - 清除 Cookie，重新登录

## 问题11：下载文件在 Downloads 里找不到

**现象：** 点击下载后，Downloads 文件夹里没有新文件。

**原因：** Chrome 下载实际上先进入缓存，然后移动到 Downloads。如果缓存中的文件移动失败或被跳过，Downloads 里就不会出现。

**解决方案：**
永远不要依赖 Downloads 文件夹来检查下载是否完成。正确的做法：
1. 点下载按钮
2. 立即运行缓存监控脚本（`wait_download_cache.py`）
3. 脚本从 `Cache_Data` 目录提取文件，这才是下载的真实位置

## 问题12：首次配置 Chrome Profile 时目录权限错误

**现象：** 运行安装脚本报错"路径不存在"或权限拒绝。

**原因：** `AppData\Local` 目录需要对应用户权限，或者路径包含特殊字符。

**解决方案：**
```powershell
# 用当前用户路径（不需要管理员权限）
$ChromeProfilePath = "$env:USERPROFILE\AppData\Local\Google\Chrome\OpenClaw"
New-Item -Path $ChromeProfilePath -ItemType Directory -Force
```

## 问题13：mcporter 工具列表为空

**现象：** 运行 `mcporter tools` 返回空，`mcporter call minimax.xxx` 报"Tool not found"。

**原因：** MCP 服务器崩溃或未正确启动。

**解决方案：**
```powershell
mcporter restart minimax
```
如果仍然不行，重启 OpenClaw Gateway。

---

## 问题14：Chrome Downloads 目录里有同名文件时下载失败

**现象：** 之前下载过同名文件，再次下载时 Chrome 自动重命名（比如加 (1)），导致监控脚本找不到正确的文件。

**原因：** Chrome 的默认行为：同名文件自动加序号。

**解决方案：**
缓存监控脚本通过**修改时间**判断最新文件，不依赖文件名，所以完全不受影响。

---

## 问题15：Chrome 关闭后重启，缓存文件消失

**现象：** 下载完成后 Chrome 崩溃或被关闭，之前缓存里的文件不见了。

**原因：** Chrome 关闭时清空了临时缓存。

**解决方案：**
缓存监控脚本在检测到文件后**立即 copy 到 media 目录**，不是 move。只要 copy 完成，即使 Chrome 关闭也不影响。脚本设计为：
1. 检测到新文件
2. 立即读取内容
3. 写入 media 目录
4. 才算完成

---

## 问题16：提示词没有触发图片生成，显示"Image Creation Disabled"

**现象：** 输入描述发送后，Gemini 没有生成图片，而是显示了文字回复或"Image Creation Disabled"。

**原因：** 提示词**没有加前缀** `Generate Image`，导致 Gemini 把它当作普通文字请求处理。

**解决方案：**
必须以 `Generate Image ` 开头（注意空格）：
```
Generate Image A beautiful sunset over the ocean, photorealistic style
```

## 问题17：提示词被 Gemini 误解为普通请求

**现象：** 发送后 Gemini 开始文字聊天而不是生成图片。

**原因：** Gemini 模式设置不对（可能选了"快速"以外的其他模式）。

**解决方案：**
确保页面底部显示的是"快速"模式（默认）。如果选了其他模式（如"深度思考"），图片生成功能可能被禁用。关闭重新打开制作图片选项。

---

## 问题18：浏览器 tab targetId 失效

**现象：** 之前获取的 targetId 无法操作，报错或无响应。

**原因：** 页面发生导航（点击链接、新对话、URL 跳转）后，原来的 tab targetId 失效。

**解决方案：**
页面导航后，必须重新执行 `browser action=open` 获取新的 targetId，然后再 snapshot。

---

## 问题19：Gemini 提示需要 Google 账号验证

**现象：** Gemini 页面显示"验证您的身份"或要求重新登录。

**原因：** Chrome Profile 的 cookie/session 过期，需要重新认证。

**解决方案：**
1. 在 Chrome 中打开 `https://gemini.google.com`
2. 按提示完成身份验证
3. 刷新页面确认工具按钮可用
4. 验证一次即可，之后会自动保持

---

## 问题20：Python requests 下载 CDN 链接成功但文件是错的

**现象：** 用 urllib 下载成功，但保存的文件打不开或图片不对。

**原因：** Google CDN 的图片 URL 有时效性，或者 URL 被截断了（分享链接里的 URL 超过一定长度会被截断）。

**解决方案：**
不要用 Python urllib 下载 CDN URL。永远用 Chrome 缓存提取，这是最可靠的方式。

---

## 问题21：PowerShell 执行策略限制

**现象：** 运行 `.ps1` 安装脚本报错"禁止运行脚本"。

**原因：** Windows PowerShell 执行策略默认禁止运行本地脚本。

**解决方案：**
```powershell
# 临时允许本地脚本（当前会话）
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope CurrentUser

# 或者使用 Bypass 策略运行脚本
powershell -ExecutionPolicy Bypass -File "安装脚本.ps1"
```
