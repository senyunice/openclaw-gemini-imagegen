# OpenClaw + Gemini 图片生成 · 完整使用指南

> 本文档详细图解从零配置到成功生图的全流程，适合首次接触的用户阅读。

---

## 一、项目简介

本项目解决什么问题？

- 让 OpenClaw AI 助手通过 Gemini 网页版自动生成图片
- 自动监控下载、自动识别文件类型、自动审核
- 全程无需人工干预，AI 自己完成

### 核心技术栈

| 工具 | 作用 |
|------|------|
| Gemini 网页版 | 图片生成引擎 |
| Chrome 远程调试 | 控制浏览器自动化 |
| Python 脚本 | 监控下载、判断文件类型 |
| MiniMax MCP | 图片内容审核 |

## 二、首次配置（只需做一次）

### 2.1 准备文件

从 GitHub 下载本项目：
```
https://github.com/senyunice/openclaw-gemini-imagegen
```

下载后，文件夹里应该有这些文件：

```
OpenClaw_ImageGen/
├── gemini-image-SKILL.md      ← AI 的操作手册
├── browser-SETUP.md           ← Chrome 配置说明
├── minimax-mcp-SETUP.md      ← 图片审核配置说明
├── wait_download_cache.py     ← 下载监控脚本（核心）
├── 问题排查指南.md             ← 常见问题和解决方案
├── 快速参考.txt               ← 速查表
└── README.md                  ← 总说明
```

### 2.2 替换用户名

本项目所有路径里包含 `%USERPROFILE%`，需要替换为你自己的 Windows 用户名。

**查看方法：**
打开文件资源管理器，地址栏输入 `C:\Users\`，里面那个文件夹名就是你的用户名。

**示例：**
- 你的用户名是 `zhangsan`
- `%USERPROFILE%\.openclaw\workspace` → `C:\Users\zhangsan\.openclaw\workspace`

### 2.3 复制 Skill 文件

将 `gemini-image-SKILL.md` 复制到：
```
C:\Users\<你的用户名>\.openclaw\workspace\skills\gemini-image\SKILL.md
```
> 如果 `skills` 目录不存在，手动创建整个路径。

### 2.4 复制下载脚本

将 `wait_download_cache.py` 复制到：
```
C:\Users\<你的用户名>\.openclaw\workspace\wait_download_cache.py
```

### 2.5 创建 Chrome 专用配置目录

打开 PowerShell，运行：

```powershell
New-Item -Path "C:\Users\<你的用户名>\AppData\Local\Google\Chrome\OpenClaw" -ItemType Directory -Force
```

### 2.6 启动专用 Chrome

在 PowerShell 运行：

```powershell
Start-Process "C:\Program Files\Google\Chrome\Application\chrome.exe" -ArgumentList "--user-data-dir=C:\Users\<你的用户名>\AppData\Local\Google\Chrome\OpenClaw","--remote-debugging-port=9222"
```

### 2.7 登录 Gemini

Chrome 打开后，访问：
```
https://gemini.google.com
```
扫码或输入 Google 账号登录。

> 以后每次使用会自动记住登录状态，不需要重新登录。

### 2.8 配置完成验证

重启 Chrome 后，确认 Gemini 页面**右上角显示你的 Google 账号**（而非"登录"按钮），说明配置成功。

## 五、每一步详细说明

### 阶段一：启动 Chrome

**为什么需要专用 Chrome？**

- 使用独立的浏览器配置，与你正常使用的 Chrome 完全隔离
- 独立配置目录，不影响日常使用
- 登录状态保存在配置目录里，重启后依然保持

**启动命令详解：**

```powershell
# 第一步：杀掉所有 Chrome（必须先杀！）
Stop-Process -Name chrome -Force -ErrorAction SilentlyContinue
Start-Sleep -Seconds 3

# 第二步：启动专用 Chrome
Start-Process "C:\Program Files\Google\Chrome\Application\chrome.exe" -ArgumentList "--user-data-dir=C:\Users\<你的用户名>\AppData\Local\Google\Chrome\OpenClaw","--remote-debugging-port=9222"

# 第三步：等 5 秒
Start-Sleep -Seconds 5

# 第四步：验证端口
Invoke-WebRequest -Uri 'http://localhost:9222/json/version' -TimeoutSec 5
```

返回 200 说明 Chrome 启动成功。

---

### 阶段二：生成图片

**关键：必须加 "Generate Image" 前缀！**

否则 Gemini 会把它当作普通文字请求，不会生成图片。

正确示例：
```
Generate Image A beautiful anime girl, silver hair, blue eyes, wearing casual outfit
```

**等待图片生成：**

- 每次发送后，Gemini 需要约 40-90 秒生成图片
- AI 每 3 秒检查一次页面
- 如果 1 分钟内没有图片，自动重新发送

---

### 阶段三：下载图片

**这是最难的部分，也是本项目的核心创新。**

**问题：**
Chrome 点击下载后，文件存在内部缓存里，文件名没有扩展名，直接读 Downloads 目录找不到。

**解决方案：**

```
Chrome 缓存目录：
C:\Users\<你的用户名>\AppData\Local\Google\Chrome\OpenClaw\Default\Cache\Cache_Data
```

下载监控脚本 `wait_download_cache.py` 自动：
1. 每 5 秒扫描缓存目录
2. 发现新文件 → 等大小稳定（下载完成的标志）
3. 读取文件头判断类型：
   - 文件头 `89 50 4E 47` = PNG 格式
   - 文件头 `FF D8 FF` = JPEG 格式
4. 自动加正确扩展名
5. 复制到 media 目录

**为什么不用 Python 直接下载 URL？**

Google CDN 图片链接包含浏览器认证，Python 请求会返回 403 错误。缓存提取是唯一可靠方案。

---

### 阶段四：审核与发送

**MiniMax MCP 审核是什么？**

用 AI 模型自动分析生成的图片内容，判断是否达标：

```
审核 prompt：
"描述这张图片的内容：人物外貌特征、服装、背景、整体风格画质，和用户需求对比是否一致"
```

**判断逻辑：**
- 图片清晰、内容完整、符合用户需求 → ✅ 发给用户
- 图片模糊/内容不符/质量差 → ❌ 重新生成
- 有问题但可修复 → 调整提示词重新生成

**重要：审核结果不给用户看**，AI 自己判断后决定是否发送。

## 六、常见问题速查

| 问题 | 解决方法 |
|------|---------|
| 工具按钮是灰的 | Chrome 未登录，先登录 Gemini |
| 1分钟还没图片 | 网络慢，重新发送 |
| 下载没反应 | 等 1-3 分钟，大文件下载慢 |
| 图片发不出去 | 检查 accountId 是否正确 |
| 下载文件打不开 | 缓存监控会自动加正确后缀 |
| MCP 报 Tool not found | 运行 `mcporter restart minimax` |

详细解决方案见 `问题排查指南.md`
